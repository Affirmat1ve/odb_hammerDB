Тестирование должно включать следующие сценарии:
- ОС: windows, arch, debian
- конфигурация: 2 core CPU + 4gb RAM, 4c+8gb, 8c+16gb, 16c+32gb

Тестирования нужно провсти 3 раза и усреднить результат.
Виртуализация через KVM, libvirt, virt-manager.

В итоге тестов худщиий результат показала ОС windows.
Её результаты:
- 2c+4gb ~1100 NOPM
- 4c+8gb ~2500 NOPM
- 8c+16gb ~6200 NOPM
- 16c+32gb ~15100 NOPM

На втором месте Debian (на форуме причину проигрыша относительно Arch - более старое стабильное ядро)
- 2c+4gb ~1550 NOPM
- 4c+8gb ~3900 NOPM
- 8c+16gb ~9200 NOPM
- 16c+32gb ~23000 NOPM

Лучшие результаты у Arch linux
- 2c+4gb ~1600 NOPM
- 4c+8gb ~4050 NOPM
- 8c+16gb ~9800 NOPM
- 16c+32gb ~25000 NOPM

Разительное отличие результатов линукса от windows может намекать на некорректную настройку последней, хотя опыт других людей подтверждает разницу в производительности.
Лично для меня вывод весьма однозначный - нет ни единой причины добровольно использовать Windows.
